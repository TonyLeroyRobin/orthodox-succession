# COUNCIL_LEADS.md — synod capture log (R5 standing rule)

Any synod named in ANY source during ANY work gets a row here — name,
date-guess, where encountered — even mid-task on unrelated work. The gap
report (`scripts/gap_report.py`) lists leads whose `cataloged` column is
empty. A lead is closed by cataloging the council (put the event ID in the
`cataloged` column) or by a maintainer out-of-scope note; never delete rows
silently.

`Lead | Date guess | Encountered in | Cataloged`. Put the event ID
(`event/council/...`) or `out-of-scope: <reason>` in the last column to
